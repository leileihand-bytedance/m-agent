# 直报写作 Skill

## 核心定位

本 skill 用于把用户提供的网页、文字或后续接入的 Word/PDF 材料，改写为面向国务院办公厅方向的信息直报初稿。

每一句规则都应能转化为写作动作。拿到素材后，按流程执行，逐项自检。

## 上一稿改稿

当底座传入 `revision: true` 时，本 skill 只处理“基于上一轮 M-Agent 生成直报初稿继续修改”的场景。

改稿时：

- 以上一稿标题、正文和来源为唯一材料。
- 按用户新的修改要求调整，例如压缩篇幅、突出政策背景、调整标题或弱化宣传表述。
- 不重新读取网页、文件或搜索资料。
- 不新增上一稿和用户要求以外的事实。
- 仍遵守直报篇幅、结构、语言和禁止事项。

## 输出范围

只输出：

- 标题
- 正文

不输出报头、期号、签发人、联系人、落款。

系统结构化结果中可以保留来源链接字段，供底座记录日志或后续系统使用，但不在用户收到的正文末尾展示。

## 篇幅要求

- 标题和正文合计以 700-800字为目标。
- 字数不够时，优先补充背景、机制、影响和下一步安排。
- 字数过长时，优先删去重复表述、宣传性形容词和次要细节。

## 结构设计

正文采用三段式思路，但不使用小标题。

### 开头

用 1-2 句说明核心事件。

可选两种写法：

- 直入主题型：近期发生的成果型事件，直接说明“做了什么 + 结果”。
- 政策背景型：长期推进的事项，先点出政策背景，再用“在此背景下，微众银行……”或“微众银行积极响应相关部署……”承接到微众银行响应和进展。

### 主体

默认围绕一条主线连贯叙述：

```text
背景/问题 -> 做法/机制 -> 成效/价值
```

不要把多个成果平铺罗列。素材中有多个成果时，默认选最重要的一个作为主线，其他内容只作为辅助。

只有当系统规划明确判断为“综合进展型”时，才允许围绕一个总主题组织 2-3 个并列板块，呈现微众银行一段时期内的综合进展，但仍要避免写成素材摘编。

### 结尾

优先使用：

```text
下一步，微众银行将……
```

默认写法不是只把眼前这件事收住，而是先写微众银行下一步怎么做，再顺势抬升到与主题一致的更高层方向，例如做好金融“五篇大文章”、服务高质量发展、提升相关领域服务能力等。

如果材料不足以支撑下一步计划，可使用克制预测：

```text
有望为……提供参考。
```

## 写作流程

### 第一步：定事件

先判断：

- 核心事件是什么，一句话能否说清。
- 能挂接什么政策背景。政策判断应优先使用共享政策研究层返回的结论，只回答“能不能挂、挂哪条、为什么、摘哪一句”。
- 是否有外部认可、监管部署、行业评价或客观数据。

### 第二步：定主线

- 默认只选 1 个核心成果作为主线。
- 主线必须外部可验证，例如上线、落地、入选、认定、数据、合作机制、政策试点等。
- 辅助信息必须服务主线，不单独展开。
- 全文必须以微众银行为组织中心，不能沿用外部素材把其他主体写成全文主角。
- 即使素材首段先写政府部门、合作方、基金、平台、活动或媒体表述，成稿也要改写为“背景一句 -> 微众银行做了什么 -> 成效 -> 下一步”。
- 单个企业受益个案不能单独撑起一篇直报；如需使用，只能压缩为主体中的一句辅助例证，不得单独成段，不得作为标题、开头或主线。
- 如果素材明显属于年度盘点、阶段性综述、综合进展类，且系统规划已标记为“综合进展型”，则主线应升级为“一个总主题”；正文可拆成 2-3 个并列板块，但每个板块都必须回到微众银行整体推进情况，不能被某个近期产品或细节带偏。

### 第三步：定开头类型

- 近期单次成果：用直入主题型。
- 长期工作且可自然呼应政策：用政策背景型。
- 案件、活动、获奖、直播、判决等时间节点稿件：默认直入主题。

不要为了显得正式而硬塞政策。

### 第四步：写标题

- 标题要准确概括核心事件。
- 能一行说清就用一行。
- 需要两行时，第一行写事件或政策语境，第二行写微众银行做法和结果。
- 如果标题分成两层意思，中间使用中文逗号或冒号连接，禁止使用空格硬断开。
- 当原始标题把其他主体放在前面时，标题要重写为“微众银行 + 核心举措/机制 + 结果/作用”，不要照搬外部媒体标题。

### 第五步：写正文

- 如果系统先给出“写作规划”，先按规划确认开头策略、主线、关键事实和关键数据，再结合材料落笔。
- 开头直接入题，不绕弯。
- 句与句之间要有自然过渡；如果前一句先写党中央、国务院、监管部署或政策背景，后一句要明确转入“微众银行积极响应”“在此背景下，微众银行……”这类响应句，不能硬切。
- 主体用逻辑链条推进，不分点。
- 段与段之间要按“背景/部署 -> 微众响应/动作 -> 成效 -> 下一步”递进，每段开头要承担承上启下作用，不能只是继续堆事实。
- 如果规划为“综合进展型”，主体可写成“总主题 -> 2-3 个并列板块 -> 阶段性成效 -> 下一步安排”，但板块之间必须有内在联系，不能简单堆素材。
- 数据只选 1-2 个最关键的。
- 数据要带时间锚点，例如“截至2026年5月”“上线一个多月来”。
- 不编造材料中没有的事实、数据、外部认可。
- 不编造材料中没有的具体政策名称；只有用户素材或补充材料明确给出政策文件名称时，才可写具体政策名。
- 结尾不只停留在就事论事；在素材允许的前提下，先写微众银行下一步安排，再自然抬升到与当前主题一致的更高层政策方向，例如做好金融“五篇大文章”，但不要脱离素材空喊口号。

### 第六步：自检

写完后检查：

| 检查项 | 要求 |
| --- | --- |
| 篇幅 | 是否接近 700-800 字 |
| 开头 | 是否直接说事，政策背景是否自然 |
| 主线 | 是否符合规划：普通稿件围绕一条主线，综合进展型围绕一个总主题组织 2-3 个并列板块 |
| 结构 | 是否避免“一是、二是、三是”和小标题 |
| 语言 | 是否去掉宣传腔和自我表扬 |
| 视角 | 是否站在国家、行业、改革或政策落地角度 |
| 事实 | 是否只使用材料可支撑的信息 |
| 结尾 | 是否具体，不喊口号 |

系统会在初稿后执行两类质量检查：

- 确定性检查：标题、字数、小标题、个案单独成段等硬规则。
- 语义审查：直报文体、主线/总主题是否清楚、主体视角、政策使用边界和宣传腔。

如发现 hard 级问题，系统会带着审查反馈自动重写一次。

## 语言原则

### 陈述事实，不评价自己

写“做了什么、结果是什么”，不要写“取得显著成效”。

可用：

- 上线
- 落地
- 服务
- 覆盖
- 支持
- 纳入
- 形成机制

避免：

- 成功打造
- 有效优化
- 显著提升
- 全面赋能
- 深度赋能
- 充分彰显

### 表达克制，不用极致词

可用：

- 有助于
- 将为
- 有望
- 进一步

避免：

- 彻底解决
- 根本破解
- 全面覆盖
- 全国领先
- 行业首创（除非材料明确给出）

### 让外部事实说话

优先使用：

- 用户本次提供的网页、文件、文字素材
- 政策文件
- 监管部署
- 合作机制
- 第三方评价
- 客观数据
- 真实案例

不要只写“受到广泛认可”。

### 站在国家视角

落脚点应是：

- 服务国家战略
- 支持政策落地
- 破解行业共性问题
- 为相关改革提供参考
- 支持小微企业、外贸、普惠金融、科技金融等重点方向

不要只写微众银行自身成绩。

## 禁止事项

- 不使用“一是、二是、三是”等序号分点。
- 不使用小标题。
- 不写宣传稿、新闻通稿、广告稿。
- 不堆砌政策口号。
- 不编造材料没有的数据。
- 不访问用户本次任务材料以外的本机文件。

## 格式规范

- 自称用"微众银行"。
- 产品名使用标准名称，如“微粒贷”“微业贷”等。
- 数字带时间锚点。
- 来源不写进正文末尾，由结构化 `sources` 字段返回。

## 常见处理

### 政策怎么挂

政策不是每篇必挂。先判断素材类型：

- 产品、机制、长期服务、政策落地类素材，可以挂一个最相关的政策背景。
- 案件、活动、获奖、直播、判决等时间节点稿件，默认直入主题，不写具体政策名称。
- 不确定是否匹配时，写宽泛政策导向，或不写政策背景。

只挂一个最相关的政策背景即可。开头或主体带一笔，不要每段都挂。

### 数据怎么用

优先使用用户本次提供素材中的 1-2 个关键数据。只有当用户素材缺少支撑主线或总主题的关键数据时，才可参考补充材料；如有冲突，以用户素材为准。

### 补充材料怎么用

政策材料或其他补充材料只用于首段背景一句或结尾落点的一笔带过，不能据此引入用户素材未出现的新业务线、新产品、新做法或新数据。

### 微众背景怎么用

直报场景下，不自动补充通用微众银行背景材料。只有当用户本次素材已经明确提到相关产品、机制或做法时，才围绕该素材本身展开，不额外引入无关的“小微服务”“普惠金融”等背景内容。

### 多成果素材怎么写

默认选最重要成果作为主线，其他成果围绕主线带一句即可。

如果素材属于年度盘点、阶段性综述、综合进展，且系统规划已标记为“综合进展型”，可以改为“一个总主题 + 2-3 个并列板块”的写法；但每个板块都必须服务同一个总主题，不能被某个近期产品、活动或细节带偏。

### 过渡怎么写

句与句之间要有承接，段与段之间要有递进。

- 如果首段先写党中央、国务院、监管部署或政策背景，下一句默认用“在此背景下，微众银行……”或“微众银行积极响应相关部署……”承接，体现“中央有部署，微众有响应”。
- 如果是直入主题型，也要先交代事件，再顺势转入微众银行做法、成效和意义，不能把素材原句硬拼在一起。
- 段首要承担承上启下作用，不能只换一组事实继续堆。

### 结尾怎么抬升

默认先写微众银行下一步怎么做，再落到与主题一致的更高层方向。

- 普惠、外贸、科创、绿色、消保等主题，都可以在最后一句自然抬升到相应政策方向。
- 如素材支撑不足，也至少要做到“下一步安排 + 宽泛方向”，不要只把全文停在某个单点动作上。
- 可自然带到做好金融“五篇大文章”、服务高质量发展等更高层方向，但不能脱离素材另起炉灶。

### 企业个案怎么用

单个企业受益案例只可作辅助例证，最多在主体段中带一句，不能单独起段，更不能把某一家企业获贷、获批、受益写成整篇直报的核心事件。

### 材料不足怎么办

如果没有网页、文字或文件材料，先追问用户补充材料，不要空写。

### 链接读不到正文怎么办

如果网页链接只能读到标题、作者名、媒体名，或根本没有提取到有效正文，应直接提示用户该链接暂时无法读取，不得继续依赖模型补写、脑补，或拿存量补充材料替代原始素材写稿。
