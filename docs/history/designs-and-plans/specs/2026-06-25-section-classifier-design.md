# 板块归位检测器（content-wrong-section）代码化设计方案

> **目标**：将 `content-wrong-section` 规则从纯 LLM 判断改为"代码关键词匹配 + LLM 兜底"的 Hybrid 架构，提高检测稳定性和准确率。

---

## 背景

当前问题：`content-wrong-section`（内容放错板块）依赖 LLM 判断，但 LLM 对板块归位规则的理解不稳定，容易漏报。

**典型漏报案例**：
- "国务院党组会议"被放在监管动态（应归党政要闻）
- "资本市场综述/A股综述"被放在监管动态（应归市场观察）

---

## 核心设计

### 板块主体关键词库

```python
# app/review/section_entities.py
```

### 检测逻辑

**识别流程（每个新闻段落执行一次）：**

```
1. 定位所属板块
   - 从当前段落往前找最近的板块分类标题（党政要闻/监管动态/同业动向/市场观察/前沿观点）
   - 记录当前段落属于哪个板块

2. 提取内容主体
   - 取新闻标题（前80字）
   - 取正文第一句（前100字）
   - 合并作为识别文本

3. 代码关键词匹配
   - 在识别文本中搜索 REGULATORY_ENTITIES / PARTY_GOV_ENTITIES / BANKING_ENTITIES 关键词
   - 匹配到哪个集合 → 推断内容属于哪个板块

4. 板块一致性判断
   - 如果匹配到 REGULATORY_ENTITIES → 期望板块 = 监管动态
   - 如果匹配到 PARTY_GOV_ENTITIES → 期望板块 = 党政要闻
   - 如果匹配到 BANKING_ENTITIES → 期望板块 = 同业动向
   - 市场观察/前沿观点：不报错（兜底板块）

5. 不一致时报错
   - 当前段落所属板块 ≠ 期望板块 → 报告 content-wrong-section
```

**边界情况处理：**
- 匹配到多个集合 → 以优先级最高的为准（REGULATORY > PARTY_GOV > BANKING）
- 未匹配到任何集合 → 不报错（市场观察/前沿观点兜底）
- 前沿观点板块内的内容 → 不报错（规则明确说明）

### Phase2 集成

`content-wrong-section` 归入 Phase2，与其他3个规则（content-out-of-scope、content-duplicate、content-outdated）并行检测：

```python
# reviewer.py - review_phase2 中新增一个步骤
def check_content_wrong_section(paragraphs, section_entities) -> list[Finding]:
    # ... 检测逻辑 ...

async def review_phase2(...):
    # ... LLM 调用 ...
    # 代码化检测（确定性高）
    code_findings = check_content_wrong_section(paragraphs, section_entities)
    # LLM 检测（处理模糊情况）
    llm_findings = await _call_phase2_llm(...)
    # 合并结果
    all_findings = merge(code_findings, llm_findings)
```

---

## 关键规则

1. **关键词匹配优先级**：REGULATORY > PARTY_GOV > BANKING
2. **兜底原则**：未匹配到任何已知主体 → 不报错（默认市场观察）
3. **前沿观点豁免**：前沿观点板块内的内容不参与板块检测
4. **保留 LLM 兜底**：代码检测后仍调用 LLM 处理边界情况，结果合并去重
