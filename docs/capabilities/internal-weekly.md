# 内参周报

## 当前能力

`skills/internal_weekly/` 是独立的内容生产 Skill，不属于审核模块，也不调用内参周报、半月报的审核代码。它只是把既有内参审核中已经确认的五类板块口径复制为自己的分类规则。

周报定位为微众银行内部管理团队和各部门的信息综述参考，选题优先服务银行经营管理。党政要闻不以是否直接出现“金融”表述作为门槛，同时覆盖宏观经济和高质量发展、科技创新和人工智能、数字经济、扩大内需和促进消费、支持小微企业和民营经济、营商环境等具有经营管理参考价值的中央重大信息。党建只纳入党中央层面的重要部署和金融监管部门自身部署，不收其他部委日常党建。

用户在写作 Bot 中发送“生成本周内参周报”即可启动，无需上传材料；发送“生成一下今天的资本市场综述”可只更新当日行情块。系统按周一出版、上一自然周统计的口径联网检索，并生成：

- Markdown 内容核对稿；
- JSON 溯源清单；
- 基于内容和来源的稳定 `draft_version`。

请求进入写作持久队列后，Bot 会先回复已经受理，再由后台完成检索和生成。处理中发生进程重启时，任务可从正式任务快照恢复；完成说明、Markdown 核对稿和 JSON 溯源清单分别保存交付检查点，已确认送达的项目不会重复发送。

初次生成不输出 Word。人工核对无误后，用户明确回复“核对无误，生成 Word 洁净版”，系统才把当前同一任务、同一 `draft_version` 的内容写入 Skill 内净化母版；这一步不重新搜索、不重新调用模型。普通“生成 Word”只会要求用户再次明确批准，资料不完整、周一收盘内容待更新、版本不一致或存在空板块时均拒绝生成。

洁净版固定保留母版的水印、红线、页眉页脚、页码、页面设置和自动目录域，使用原有标题样式写入五个板块。程序先清除上一期目录缓存，再把任务副本放到 Microsoft Word 自有沙盒目录，由本机 Word 在后台按最终分页更新整个目录并保存。写回任务输出前还会逐项核对目录标题、正整数页码、`PAGEREF` 和 `_Toc` 书签；任何一项不完整都停止交付。用户收到的洁净版打开后已有完整目录项和页码，不需要再手工更新。洁净版不含原文链接、核验原句、正文哈希、草稿版本和待核说明，前沿观点正文中的报告出处继续保留。

## 固定内容

板块固定为党政要闻、监管动态、同业动向、市场观察、前沿观点。

完整周报在同一后台任务内并行执行这五个板块，每个板块独立检索、读取、筛选和校验，最多同时运行 5 个板块任务；全部返回后再按固定顺序汇总。某一板块失败不会取消其他板块，但会在核对稿中明确标为待核，并使整期结果不可批准。该机制不会把一份周报拆成五个企业微信任务；单独生成今日资本市场综述时仍只运行市场观察更新链路。

四个普通信息板块分别检索、分别筛选，不再混用一个候选池。党政、监管、同业和市场材料必须通过目标板块与代码分类双重一致校验；无法识别的内容直接排除，不默认落入市场观察。党政要闻在多个模型批次汇总后还会执行事件级取舍：同一有专名的大会、论坛、峰会、年会或博览会最多合并 3 篇互补正式稿，分别保留重要讲话、行动部署、组织或机制成果及其来源；入选后再综合为一个总标题和一个连贯段落，不按来源机械拼段。重复评论、侧记、反响和泛泛回顾不再单列。陆家嘴论坛等后续活动沿用同一通用规则，不逐个写死。核对稿始终显示五个板块，空板块会显式标注，不再因为没有内容而消失。

候选链接还会按目标板块再次套用信源表。党政要闻固定登记中国政府网“要闻列表” `https://www.gov.cn/yaowen/liebiao/` 作为发现入口，直接读取页面使用的官方 JSON 列表，先按统计期、经营管理相关主题和 `gov.cn` 精确主机筛选，再读取文章正文；同时围绕宏观发展、科技创新/人工智能、扩大内需/促消费/小微企业执行公开搜索，并由新华社、人民网补充中央重大部署和党建材料。固定列表与搜索候选合并去重后进入相同的模型筛选和代码复核。

监管动态改为按机构分组采集。人民银行固定读取新闻发布列表；金融监管总局固定读取“监管动态”“政策解读”“领导活动及讲话”三个栏目背后的官方列表和正文接口；证监会固定读取“证监会要闻”页面使用的官方列表接口。每个机构在统计期内只要取得至少一篇通过域名、日期和正文校验的原文，就不会再执行该机构的公开检索；没有合格正文时才只对该机构检索。外汇局当前未登记固定栏目，保留官网优先的公开检索。监管党委、党建关键词已并入四个机构各自的查询，不另做跨机构泛搜。公开检索兜底可以使用登记的主流财经及中央媒体报道，但仍须明确监管主体、动作、日期和原文证据。条目标题、摘要和核对稿来源机构统一使用人民银行、金融监管总局、证监会或外汇局等实际监管机构；国新办主持发布会时不再把“国新办举行新闻发布会”当成内容主体，但原始网页题名和证据句仍保留在溯源记录中。总局后台正文接口只用于取得内容，核对稿和溯源清单始终保存人工可打开的官方文章页。中央政府门户 `gov.cn` 只允许精确主机，地方 `*.gov.cn` 页面不会再被当成党中央或国务院信源。固定列表记录、搜索链接、正文地址和网页跳转后的最终链接都要通过同一检查。

登记来源若出现一次临时网页读取失败，系统会在同一 URL 上重试一次；仍失败才记录待核，不转向未知来源。中国政府网文章页即使采用公共页头提前闭合 HTML 的非标准模板，网页工具也会切换解析方式恢复后续正文和发布日期，避免只把“首页、登录、无障碍”等导航交给模型。普通候选按最多 5 页一批筛选，模型必须逐条返回入选或不入选判断；如果整批返回空判断，系统会自动重试一次，连续空答或结构化输出失败时会连同本批候选标题一起记录，避免材料无痕消失。某一批失败不会再让整个同业或监管板块归零；超长正文会在保留原链接和原文校验的前提下限制进入模型的长度。

普通内容允许 AI 对原文进行概括和压缩，证据机制不要求摘要逐字复制原文。程序把正文切成编号证据块，模型选择最多 3 个编号，程序再直接取回原文；如果摘要中的数字、单位、政策题名或增减方向发生变化，系统只允许基于同一原文校正一次。校正后核心意思仍不能由证据支持时才排除。核对稿展示压缩后的摘要和原文证据，便于人工判断是否准确；洁净版继续只保留批准后的摘要。

同业动向首批覆盖境内民营/数字银行、国际及香港数字银行、国内银行科技子公司三组。国际组包括 Monzo、Starling、Revolut、N26、Nubank、KakaoBank、ZA Bank、Mox Bank；科技子公司组首批包括建信金科、工银科技、兴业数金、招银云创。微众银行自身不作为同业。已核对的 5 期案例均没有同业动向板块，因此本栏目不照搬案例条数，而是按周报服务内部管理团队的定位独立建立规则。

三类机构名单按每组最多 5 家分片，分别检索经营业绩与战略、产品科技与风险两类信息；各组通过域名、日期和正文校验后轮询合并。信源分为机构官网/投资者关系/年报、监管与交易所法定披露、权威媒体补充三级。同业条目按可比相关性、变化重要性、战略信号、证据质量、经营启示五项各 0 至 2 分评分，总分不低于 7 分才可入选；最多 5 条且不凑数。普通营销促销、获奖、一般会议、招聘和没有实质业务变化的宣传稿排除。实体别名、官方栏目、母行补充域名和查询模板由 `skills/internal_weekly/references/source-registry.yaml` 统一维护，完整细则见 `skills/internal_weekly/references/peer-activity-system.md`。

市场观察第一项固定为“资本市场综述”，必须依次包含：

1. 上周 A 股；
2. 出版日周一 A 股收盘；
3. 上周港股；
4. 上周美股。

每组必填指数和计算口径由代码控制。来源直接披露本期涨跌幅时，系统保留该数值和原句；模型若同时多填起止收盘值，代码清空多余字段，不混用两种模式。来源只披露起止收盘值时，涨跌幅由代码计算。周一 15:00 收盘前仍会生成上一周内容，但跳过周一 A 股行情检索，在固定位置用红色粗体显示“今日资本市场内容待收盘后更新”，避免把盘中数据当作收盘数据；该稿不可批准。15:00 后再次生成，系统才补齐周一收盘段落。

单独更新指令在收盘前不做任何搜索，只返回醒目占位；收盘后只抽取 `monday_a` 三个当日 A 股指数，输出独立的“今日资本市场综述更新”核对稿和溯源清单，不重复搜索周报其他板块，也不覆盖整期核对稿。

市场数据按“上周 A 股”“周一 A 股”“上周港股”“上周美股”四组分别抽取，降低单次结构化输出复杂度；四组完成后仍由代码统一合并，任何必填指数缺失都不会生成伪完整综述。市场结构或驱动因素背景句是可选项，证据无法逐字核对时只排除该句并提示，不会连带丢弃已通过硬校验的必填指数综述。

对于来源直接披露、且标题或证据句明确写明“上周、本周、一周、单周、周涨、周跌”的周度涨跌幅，代码使用本次任务的统计周和该周最后一个工作日，不再采用模型误填的周评文章发布日期。只有起止收盘值或缺少周度语义时仍按来源日期硬校验，不会用统计周掩盖证据缺口。英文来源按固定别名回源定位 Dow/Dow Jones、Nasdaq、S&P 500 等真实原句，最终仍由代码输出统一中文指数名；数值不一致时不会因为别名匹配而放行。

行情检索会带上明确统计期、出版日和指数名称，并使用自然语言查询，不使用当前 DeepSeek 原生搜索无法稳定识别的 `site:`/`OR` 组合。同一市场组允许多条中性查询合并去重；周度查询使用统计期内最后一个工作日，降低只命中当日行情或因周日日期而漏检周评的概率。网页或模型返回中文年月日、斜杠日期等常见格式时，系统先归一化，再执行同一套交易日和出版日硬校验。

真实网页可读性校验后，上周 A 股和美股会补充检索新华财经全球周评，美股增加英文日期的美联社查询，港股继续使用能同时列出三项必填指数的周评来源。搜索结果只有空摘要时不能作为证据；必须由网页读取器取得正文并通过逐字校验。

资本市场综述只是市场观察的固定首项，不是整个板块。对 5 期案例逐项归纳后，非固定内容实际涉及国内利率与贷款定价、全球主权债和通胀、主要央行、海外市场异常波动、地缘贸易航运对油价和黄金的传导、标志性 IPO，以及银行理财和资管结构变化；案例中以监管处罚为核心的内容按本 Skill 的互斥规则归监管动态，不在市场观察重复收录。

系统因此改为七组独立查询：国内宏观与经济运行、国内货币信用利率汇率、财富管理与资产管理、全球宏观与主要央行、全球市场波动与金融稳定、地缘贸易能源与供应链、重大资本市场事件。每条查询最多读取 10 个候选，各主题通过域名、日期和正文校验后轮询合并，避免热门主题垄断候选池。模型按影响范围、变化幅度、银行相关性、证据质量和本周新颖性五项各 0 至 2 分评分，必须说明“事件或数据—影响渠道—对银行经营或市场判断的意义”；总分不低于 7 分才可入选，非固定条目最多 5 条、不设最低数量。事实相当时优先官方原始来源，权威媒体用于补充跨市场影响；普通日评、轻微涨跌、传闻、机构宣传和固定综述已覆盖的常规行情不收录。完整规则由 `skills/internal_weekly/references/market-observation-system.md` 持有。

前沿观点当前采用“研报采集、中文压缩、原文核验”。系统使用多条不写死报告题名的中性查询并合并去重，除 Working Paper、Bulletin 外，还按统计年份和月份检索 BIS Annual Economic Report、数字支付与银行金融报告系列，并扩大每条研究查询的候选结果数，降低搜索排序波动。系统先找上一完整自然周内的报告；周内没有合格结果时，补充检索截至统计期末近 30 日内的报告，并明确标记使用兜底。出版日当天新发布的报告不进入上一周周报；模型返回日期还必须与来源页面日期一致。模型依据入选原文生成中文题名和中文压缩摘要，过长内容按完整句边界进一步压缩，正文结尾标明中文机构名和报告原文题名。用于支撑摘要的段落必须逐字存在于已读取页面并以完整句末标点结束；大段英文不再直接作为周报正文，而是保留在核对信息和 JSON 溯源清单中供人工核验。近 30 日仍没有合格报告时保留缺口，不生成 AI 观点替代。

## 数据源调研结论

公开来源确实存在，但“页面可访问”与“适合稳定自动采集”是两回事。当前分层如下：

| 用途 | 已核验的公开来源 | 当前可用方式 | 主要边界 |
|---|---|---|---|
| 金融监管动态 | [人民银行新闻发布](https://www.pbc.gov.cn/goutongjiaoliu/113456/113469/index.html)、[金融监管总局监管动态](https://www.nfra.gov.cn/cn/view/pages/ItemList.html?itemPId=914&itemId=915&itemUrl=ItemListRightList.html&itemName=%E7%9B%91%E7%AE%A1%E5%8A%A8%E6%80%81)、[政策解读](https://www.nfra.gov.cn/cn/view/pages/ItemList.html?itemPId=914&itemId=917&itemUrl=ItemListRightList.html&itemName=%E6%94%BF%E7%AD%96%E8%A7%A3%E8%AF%BB&itemsubPId=916)、[领导活动及讲话](https://www.nfra.gov.cn/cn/view/pages/ItemList.html?itemPId=914&itemId=919&itemUrl=ItemListRightList.html&itemName=%E9%A2%86%E5%AF%BC%E6%B4%BB%E5%8A%A8%E5%8F%8A%E8%AE%B2%E8%AF%9D)、[证监会要闻](https://www.csrc.gov.cn/csrc/c100028/common_xq_list.shtml) | 先读取栏目自己的列表或数据接口，按统计期发现原文，再逐篇读取正文 | 固定入口按机构独立判断；该机构无合格正文时才公开检索，外汇局仍为官网优先检索 |
| 同业经营与创新 | [Monzo投资者信息](https://monzo.com/investor-information)、[Starling投资者信息](https://www.starlingbank.com/investors/)、[Revolut财务报告](https://www.revolut.com/reports-and-results/)、[Nubank投资者关系](https://www.investidores.nu/)、[Mox媒体中心](https://mox.com/media/)、[香港金管局认可机构登记](https://vpr.hkma.gov.hk/eng/regulatory-resources/registers/register-of-ais-and-lros/) | 官网和投资者关系确认经营、产品和战略事实，监管/交易所披露交叉核验牌照、财务和治理，媒体补充背景 | 官网营销稿不因来源官方自动入选；仍须达到同业价值评分和逐字证据要求 |

同业首批官方入口已经登记在 Skill 自有信源表。国际数字银行的官方页面可持续提供公司新闻、产品、经营和监管牌照动态；国内银行科技子公司的官网可提供产品、项目、技术和经营信息。自动筛选仍坚持“机构官网优先，权威财经媒体补充”，新闻稿只有同时满足统计期、经营管理相关性和原文证据要求才入选，纯营销活动不收录。

因此第一阶段采用“官方指数机构/交易所优先，主流财经媒体公开 HTML 作为补充”的方法，并要求每个数值都带原页面证据。若需要每周稳定无人值守，下一阶段应优先评估机构现有的 Wind、Choice、同花顺 iFinD 等已授权数据服务，而不是继续增加不稳定网页抓取；是否接入取决于现有许可、API 权限和合规要求，本阶段不预设采购。

搜索引擎返回的链接会在打开网页前经过来源白名单筛选，避免无关地方站点、招标页和未知来源拖慢或污染结果；读取失败时，用户侧只看到失败来源域名，底层连接错误保留在内部日志范围。

国内智库、券商和银行研究报告可以作为后续候选，但常见登录、付费、版权或 PDF 访问限制。在完成来源授权和真实样本验收前，不把这类站点承诺为稳定自动数据源。

## 人工核对方式

核对稿为每条内容显示原文链接、机构、发布日期、核验原句；研报项额外显示报告位置。网页原文逐字校验只忽略零宽字符、软连字符和空白排版差异。行情模型证据句有轻微措辞差异时，系统必须用固定指数名称和相同数值在网页中重新定位真实分句，并保存该真实分句；名称或数值不一致仍会失败。溯源清单保存网页正文哈希，方便确认核对时看到的来源内容未被静默替换。

只有党政要闻、监管动态、同业动向均有合格条目，资本市场综述已补齐周一收盘数据且前沿观点有合格中文摘要和逐字研报证据，整期结果才标记为“可提交人工核对”。周一收盘前的占位稿可以提前查看上周内容，但保持不可批准，也不会自动生成洁净版。

人工批准指令与当前任务绑定。系统同时核对核对稿和任务元数据中的出版日、统计期、`draft_version` 与 `ready_for_approval`，任一不一致即停止。Word 期号以批准母版的 `2026-07-20=2026年第28期（总409期）` 为基准按周递增；用户明确填写的年度期号和总期号优先，跨年度必须明确提供。
