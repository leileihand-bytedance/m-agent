# 简报写作

## 定位

简报面向深圳市地方政府和监管部门，展示微众银行近期动态、成果和业务价值。当前由一套统一简报工作流自动处理单素材和多素材场景。

业务规则唯一来源：

- `skills/writer1/SKILL.md`

`writer1` 是当前唯一业务规则、模型提示词、权限和队列 ID。单素材和多素材都进入同一工作流，由实际材料数量和关系决定处理模式。

## 输入和路由

- 支持文字、网页链接、Word、PDF 和 PPTX。
- 一份核心材料采用单素材模式，多份材料先判断关系后采用多素材整合模式或建议拆分。
- 文件和链接数量不直接决定文章结构：多份同一事件材料先去重和收束，一份文档中的多个明确子素材也可以被拆分后整合。
- 历史持久化结果中的退役 ID 在读取时归一为 `writer1`，旧稿仍可继续修改。
- 用户可以先说明写简报再连续发送材料，也可以先发材料后说明用途。

## 当前写作边界

- 正文首次出现主体时使用“深圳前海微众银行（以下简称“我行”）”，后文使用“我行”。
- 正常篇幅约 1000 字，最长不超过 1200 字。
- 写作前先从九类简报类型中选择对应写法，形成核心信息、受众价值、1 至 3 个主体板块和可追溯事实台账；多素材还要确定关系、主次和取舍。
- 活动亮相、外部认可等信息密度较低的题材仍可正常写作，但会压缩会务、名单和宣传性内容。
- 政策库和微众银行信息库只提供少量高相关背景，不能代替用户事实材料。
- 初稿生成后检查最长篇幅、数字依据和高风险强结论；确定性检查不替代人工审阅。
- 继续对话改稿时会解析修改范围；只改标题或指定段落时，未点名内容保持不变，并进入平台版本链。
- 用户可以在首次写作要求中加入“输出 Word”或“生成正式文档”，也可以对已完成简报直接说“把这篇简报导出 Word”。只导出时复用上一稿，不调用模型重写内容。
- 期号、日期、责任编辑和联系电话可以在首次要求或后续对话中修改，例如“日期写7月21日，期号第18期，责任编辑张三”。这些设置随简报版本保存，下一轮只改其中一项时保留其他设置并重新生成 Word，不调用写作模型。签发人固定为母版值，不支持修改。

质量回归方法见 [简报质量回归方法](brief-quality-regression.md)。

## 输出

默认返回标题、正文和来源。用户明确要求 Word 或正式文档时，系统先返回同一标题和正文，再发送 `.docx` 附件。成品使用项目内受控母版，替换标题、按自然段拆分的正文及用户明确设置的期号、日期、责任编辑和联系电话；签发人、红线、页码和其他页面版式保持母版原样，不写入来源、日志或审核提示。
