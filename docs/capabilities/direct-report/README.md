# 直报写作

## 定位

直报根据用户提供的网页链接、文字、Word、PDF 或 PPTX 材料生成报送初稿。业务规则唯一来源是 `skills/direct_report/SKILL.md`。

## 入口和流程

```text
企业微信写作 Bot / 本地入口
  -> 写作材料组装
  -> direct_report
  -> 读取用户材料
  -> 按需检索政策或公开背景
  -> 生成结构化标题、正文和来源
  -> 确定性检查与语义质检
  -> 返回初稿并建立版本链
  -> 用户明确要求时，按正式母版生成并返回 Word
```

新初稿使用写作持久队列；用户收到初稿后，可以继续提出压缩、改标题、调整重点等改稿要求，也可以直接说“导出 Word”。用户只要求导出，或只补充期号、总期号和日期时，系统沿用当前稿件，不重新调用模型改写正文；生成结果会在企业微信中同时交付标题正文和 Word 附件。

## 当前边界

- 全文主体直接使用“微众银行”，不使用全称加简称，也不使用“我行”。
- 政策背景必须与材料类型和主题直接相关，不能为了正式感硬挂政策。
- 具体政策名称、数字和事实必须来自用户材料或可追溯来源。
- 部分链接读取失败时，用户可以选择使用其余材料继续，或补充正文。
- critic 默认提供风险提示，不因语义意见自动重写，避免增加生产失败概率。
- Word 使用批准母版原位替换标题和正文；期号、总期号和日期只按用户明确要求填写，未提供的字段保留占位符。来源、日志和审核提示不进入成品。
