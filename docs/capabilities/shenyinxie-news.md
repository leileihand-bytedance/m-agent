# 深银协动态

## 定位

深银协动态根据用户指定的月份和上半月/下半月，搜索权威媒体中的微众银行正面报道或成果，筛选后生成 Word。业务规则唯一来源是 `skills/shenyinxie_news/SKILL.md`。

## 使用要求

- 用户必须明确月份和上半月或下半月。
- 信息不完整时先追问，确认前不发起搜索。
- 搜索只建立候选池，最终入选还要核验原文域名、发布日期、正文和报送价值。
- 搜索覆盖普惠、小微、科技创新、消费者权益保护、金融教育、反诈、社会责任、公益、乡村振兴、绿色金融、党建、合作、获奖和国际化等主题，但主题命中不等于自动入选。
- 每条查询执行三次并合并，以重复命中次数、标题/摘要相关度和媒体等级做确定性排序后再截取候选；首轮另设主流财经媒体、AI/算力/智能体和专利/知识产权通用检索入口，降低同一期偶发少搜或候选被噪声挤出的概率。
- 优先采用微众银行主流媒体专题全文；不足 3 篇时扩大到已核验行业、广东和深圳主流媒体，前两轮零专题全文时才启用经核验补充媒体。首轮已经发现的补充媒体候选会保留到第三轮判断，不会因当轮媒体等级不同而提前丢失。
- 综合报道只能按原文顺序摘编，并在 Word 中标注原报道标题、链接和摘编说明。
- 模型已认定为唯一核心主体、内容正面且标题适合报送的专题稿由代码统一按全文采用，避免同一篇稿因模型模式波动时而全文、时而摘编。
- 负面或疑问式标题不能直接全文报送；只有原文中微众银行正面行动与成果可独立成立时，才逐字段落摘编并改为准确标题。
- 网页读取降级产生的标题、来源、规范链接和日期会被安全恢复，YAML 元数据和 `.md` 代理尾缀不会进入正文或原文链接；无法恢复标题时拒绝候选。
- 只讲分红、派现、利润分配或股东回报的报道不作为微众银行正面成果采用。
- 核心事件为微众银行高管到其他机构任职、微众成果仅作为个人履历背景的报道不采用，也不摘取履历数据包装成动态。
- 只要找到 1 至 3 篇合格专题全文，就只输出这些专题全文；只有一篇合格专题全文都没有时才考虑综合稿摘编，摘编稿总数最多 2 篇。
- 无稿时区分没有搜索结果、媒体范围不符、原文不可读、日期或正文核验失败和报送价值不达标，不再统一显示为完全没有内容。

## 输出和边界

- 输出复制经确认并已净化的静态 Word 模板，不读取桌面案例正文。
- 生成副本只做 macOS 中文字体兼容映射，不修改母版、页边距、字号、行距和段落结构。
- 入选稿统一转为简体中文；可确定识别的站点尾部推广区会被移除。
- 正文开头的“【××报道】”“【××讯】”等信源标记单独显示为红色，方便用户人工删除，其后正文仍为黑色。
- 原文链接以可点击超链接写入文档。
- 搜索、网页读取和模型均经过底座受限工具。
- 当前不支持生成后继续多轮改稿，也尚未进入写作持久队列。
