# 审核共享核心实施计划

## 当前批次

第一批迁移纯文字、通用Word和静态HTML，保持现有规则、模型调用计划、输出、标记和任务交付兼容。

实施步骤：

1. 建立规则目录、规则矩阵和静态审核profile。
2. 抽取共享问题模型、模型结果解析、调用指标、固定预算重试、逐字证据和去重原语。
3. 将纯文字、通用Word和HTML显式接入各自profile。
4. 保留 `reviewer.py`、`review_metrics.py` 等旧导入路径的兼容出口。
5. 运行共享核心、通用审核、HTML、任务执行、入口、内参、半月报、PPT、公文格式和多文件离线回归。
6. 合并到 `main` 后重启生产审核Bot，使用新文字和新文件做小范围验收，不重复全量运行旧质量基线文件。

## 后续批次

第二批迁移内参和半月报的模型运行、证据、去重及指标，专属提示词和结构判断保持原位。第三批只让PPT复用合适的模型运行、证据协议和指标；公文格式只复用结果与指标，多文件只复用双边证据和运行原语。

## 合并条件

- 规则profile不能启用其他审核类型的专属规则。
- 规则ID、问题描述、证据、标记位置和返回形式保持兼容。
- 相同测试场景的模型调用次数不增加，重试预算不扩大。
- 全量离线测试和文档闸门通过。
- 真实模型测试只在合并后的生产目录按现有配置执行；任务分支不得读取生产密钥。
