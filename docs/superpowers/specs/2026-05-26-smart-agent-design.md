# 智能体风格沉淀系统设计

> 日期：2026-05-26
> 版本：v0.1

## 1. 目标

将 M-Agent 从"规则驱动的机械工具"升级为"智能体"，能够：
- 理解用户意图，不只是匹配规则
- 根据上下文智能推断下一步
- 从用户反馈中学习迭代
- 实时响应用户消息

## 3. 模块职责

### 3.1 Intent Classifier（意图分类器）

**职责：** AI判断用户消息的意图类型

**意图类型：**
- `raw_material` - 原材料，需要提炼
- `conclusion` - 结论/观察，可直接写入
- `command` - 指令（如开始提炼、确认）
- `question` - 询问
- `unknown` - 未知，需要追问

**判断逻辑：**
- 原材料：描述事件、提及领导言论、包含具体场景
- 结论：表达偏好、风格建议、使用"要"、"不要"、"偏好"等词
- 指令：包含"开始提炼"、"确认"、"不入库"等关键词
- 结论类消息无领导标识 → 需要追问"谁的风格？"

### 3.2 Context Manager（上下文管理器）

**职责：** 维护当前会话状态

**维护信息：**
- `current_leader` - 当前指定的领导（01-04映射）
- `pending_materials` - 待处理的材料列表
- `last_interaction` - 最近一次交互时间和内容
- `conversation_context` - 最近几轮对话的摘要

**行为：**
- 消息无领导标识时，检查上下文推断
- 上下文丢失时，主动询问用户

### 3.3 Agent Core（智能体核心）

**职责：** 根据意图执行相应动作

**原材料处理流程：**
1. 检查是否有领导标识
2. 无标识 → 询问"谁的风格？"
3. 有标识 → 保存材料到 `source/`
4. 调用 AI 提炼
5. 返回确认请求给用户

**结论处理流程：**
1. 检查是否有领导标识
2. 无标识 → 询问"谁的风格？"
3. 有标识 → 直接生成确认请求
4. 用户确认 → 写入 `profile.md`

**指令处理流程：**
- `开始提炼` → 检查有无疑材料，执行提炼
- `确认 X` → 写入 profile
- `不入库` → 记录到学习记忆，不写入

### 3.4 Profile Manager（档案管理器）

**职责：** 管理领导风格档案的读写

**文件结构：**
```
data/leaders/{领导标识}/
├── profile.md      # 风格档案
├── source/         # 原始材料
├── suggestions/    # AI提炼建议
├── update-log.md   # 更新记录
└── memory/         # 学習记忆
    └── preferences.json  # 偏好记录
```

**偏好学习机制：**
- 用户拒绝建议 → 追加到该领导的"慎用模式"
- 下次生成建议时，AI自动避开这些模式
- 记录格式：
  ```json
  {
    "rejected_patterns": ["过于乐观", "频繁变化"],
    "rejected_suggestions": [具体建议内容]
  }
  ```

## 4. 领导标识映射

**配置文件：** `data/leader-mapping.json`

```json
{
  "01": "张总",
  "02": "李总",
  "03": "王总",
  "04": "刘总"
}
```

**使用方式：**
- 用户发送 `01 今天会上强调...` → 识别为张总的材料
- 用户发送 `01` → 识别为指定张总
- 用户发送 `01 确认` → 对张总执行确认

## 5. 自迭代机制

### 5.1 学习来源

1. **拒绝反馈**
   - 用户发送"不入库" → 记录被拒绝的建议模式
   - 用户说"第2条不对" → 记录具体建议内容

2. **修改反馈**
   - 用户修改AI建议 → 记录修改前后的差异

3. **确认反馈**
   - 用户只确认部分 → 记录被跳过的建议特征

### 5.2 学习应用

生成建议时：
1. 读取该领导的 `preferences.json`
2. 构建提示时，包含"避免这些模式"的指示
3. AI生成时自动避开已知的拒绝模式

## 6. 消息格式

### 6.1 用户发送

| 格式 | 示例 | 含义 |
|------|------|------|
| `{领导} {内容}` | `01 今天会上强调服务小微` | 材料，指定领导 |
| `{内容}` | `材料要有数据` | 结论，无领导 |
| `{领导} {指令}` | `01 开始提炼` | 指令 |
| `{领导}` | `01` | 指定领导 |

### 6.2 AI回复

**原材料提炼确认：**
```
已收到"张总"的材料，分析如下：

本次观察到的风格倾向：
1. 强调服务小微
2. 注意风险防控
...

请回复：
- 确认全部
- 确认 1、3
- 不入库
```

**结论写入确认：**
```
将以下内容写入"张总"档案：
- 材料要有数据

请回复：
- 确认
- 取消
```

**追问：**
```
这是谁的风格？请回复（如：01）
```
