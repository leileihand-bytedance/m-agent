# M-Agent 本机管理后台

本文档只说明管理后台当前已经具备的能力、运行方法和安全边界。未完成的后台功能统一记录在 `docs/development/TODO.md`。

## 定位

管理后台是仅供本机使用的项目观察和配置工具，不是面向企业微信用户的业务入口，也不是多人管理平台。

默认监听：

```text
127.0.0.1:8787
```

不要直接暴露到公网或公司内网。

## 启动

```bash
uv run --locked python -m app.admin.server --port 8787
```

浏览器访问：

```text
http://127.0.0.1:8787/
```

## 当前功能

### 项目总览

首页以五层架构关系图作为“项目总览”：

```text
用户入口
  -> 通用底座
  -> 业务功能
  -> 工具与知识库
  -> 运维与数据
```

可在两个视图间切换：

- `关系图`：查看节点之间的调用、材料提供、状态上报和结果回传关系。
- `状态清单`：按五层结构扫描全部节点及其状态。

关系图支持缩放、拖动、适应画布、节点筛选和详情查看。它使用仓库内固定的 `vis-network 10.1.0`，不依赖 CDN；脚本和许可证位于 `app/admin/static/vendor/`。

### 状态口径

管理后台不调用大模型推断建设状态，也不维护第二套项目进度。页面状态只组合以下可核对事实：

- `skills/*/config.yaml` 的启用状态。
- `docs/development/TODO.md` 中仍开放事项的状态和下一动作。
- 对应代码模块是否存在。
- 写作、审核、材料润色和运维 Bot 的心跳。

建设状态包括：

- `稳定运行`：当前能力已作为稳定基础使用。
- `已上线·优化中`：主体能力可用，但仍有开放的质量或体验待办。
- `建设中`：关联待办正在进行。
- `待建设`：关联待办未开始，且没有已实施证据。
- `已暂缓`：关联待办明确暂停。
- `已关闭`：Skill 已安装但当前关闭。

建设成熟度和运行在线状态互相独立。例如“已上线·优化中 / 心跳超时”表示功能已经存在，但当前 Bot 进程需要处理。

业务能力节点还显示“持久队列”或“实时执行”。该标记直接读取写作和审核 handler 的真实注册表：深银协动态与八类审核显示为持久队列；综合调研、内参周报和即时润色显示为实时执行。迁移代码与管理台标记因此使用同一事实来源，不需要另维护一份清单。

架构节点、明确关系、职责和代码证据维护在 `app/admin/services.py`。新增、删除或改名节点时，必须同步关系表和测试。

### 板块信息

管理后台展示底座、写作、审核、知识库、入口运维和管理后台六个板块的当前事实、最近更新和首要开放待办。政策库和银行信息库当前只展示汇总数量，不展示材料正文。

### 审核模块统计

审核对外仍是一个能力，但管理后台按八类内部子能力独立展示：通用文字、通用 Word、静态 HTML、内参、半月报、公文格式、PPTX 和多文件联合审核。每类分别汇总任务总数、完成、失败、进行中、已交付、交付失败、平均处理耗时、模型调用/失败和问题数量。

统计只读取任务 `meta.json` 中不含正文的能力身份与运行指标，以及 `status.json` 中的处理和交付状态。管理台不会为统计读取审核结果、标注文档或用户材料。历史任务优先按任务类型归类；没有可靠类型信息的旧归档仍计入审核总量，但不猜测所属子能力。

### Skill 开关

后台读取并修改：

```text
skills/<skill_id>/config.yaml
```

只允许修改 `enabled: true/false`。开启 Skill 只表示它进入路由候选；实际执行仍要求 workflow 可用，且用户权限允许。

### 用户权限和任务记录

用户权限和最近任务记录默认不读取、不展示，也不会进入默认页面源码。管理员主动打开以下地址后才加载：

```text
/?show_sensitive=1#users
```

用户权限来自本机文件：

```text
config/platform-policy.yaml
```

最近任务来自统一非 Git 数据根目录，只展示必要摘要，不展示用户原文、附件内容、模型提示词或密钥。写作和审核板块保留任务数量汇总，但只读取状态及元信息文件。

### Bot 心跳

页面读取四个 Bot 的心跳用于本机观察：

- 写作 Bot
- 审核 Bot
- 材料润色 Bot
- 运维 Bot

控制台能显示某个服务，不代表运维 Bot 已对该服务配置掉线告警；告警范围仍以运维配置为准。

## 安全边界

- 默认只绑定本机地址。
- 不读取或展示 `.env`、API Key、Bot Secret。
- 不提供任意文件浏览、任意 SQL 或任意命令执行。
- Git 查询是代码内固定的只读命令，不接受页面传入命令，也不自动联网。
- 页面只允许写 Skill 开关和用户权限；项目总览、TODO、运行状态和更新记录均为只读。
- 动态文字必须转义，避免 Skill、TODO 或 Git 摘要被当作 HTML 执行。

如果以后改成多人后台，必须先完成登录鉴权、操作审计、CSRF 防护、细粒度权限和内网部署方案。这些需求在实施前只进入 TODO，不写入本文的当前能力。

## 代码边界

```text
app/admin/                         # 后台实现
tests/test_admin_services.py       # 数据服务测试
tests/test_admin_server.py         # 页面和接口测试
docs/operations/admin-console.md   # 当前运行与操作说明
```

管理后台只负责配置和观察，不能直接实现写作、审核或知识库业务逻辑。

## 验证

```bash
uv run --locked pytest tests/test_admin_services.py tests/test_admin_server.py -v
uv run --locked python scripts/project_docs.py check
```

测试至少覆盖：

- 五层架构和关系引用完整。
- 仓库内每个已安装 Skill 都映射到架构节点。
- TODO、Skill 状态、代码证据和心跳能形成正确状态。
- 页面只加载本地关系图库，不依赖 CDN。
- Bot 心跳缺失、超时或损坏时安全降级。
- 默认页面不调用用户权限和任务读取函数。
- 动态文字被正确转义。
- 页面不泄露密钥或用户材料正文。
- 八类审核子能力始终独立展示，任务类型回退归类、处理/交付状态和平均耗时口径正确。

政策库可视化管理等尚未实现的后台能力见 `docs/development/TODO.md`，不能根据本文或页面占位说明为已完成。
