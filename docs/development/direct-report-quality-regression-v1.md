# 直报质量回归测试集 v1

本文档记录直报写作质量回归测试的使用方式。目标是让后续优化 prompt、政策库、网页读取、改稿能力时，有一组固定样本可以反复对比，而不是只凭单次感觉判断。

## 样本来源

当前 v1 样本来自用户提供的 4 个公开网页链接：

1. 百家号：`https://baijiahao.baidu.com/s?id=1869477855753615781&wfr=spider&for=pc`
2. 中国证券报：`https://www.cs.com.cn/yh/04/202306/t20230614_6350212.html?bsh_bid=5960157538`
3. 第一财经：`https://www.yicai.com/news/102917116.html`
4. 同花顺财经：`http://news.10jqka.com.cn/20250527/c668479924.shtml`

样本配置文件：

```text
tests/fixtures/direct_report_quality_cases.json
```

自动化测试文件：

```text
tests/test_direct_report_quality_regression.py
```

## 自动化测试覆盖什么

自动化测试只覆盖稳定、可机器判断的部分：

- 4 个用户提供链接是否都进入回归集。
- 每个样本是否能被路由为 `direct_report`。
- 每个样本是否有人工评测字段。
- 已知坏样本是否能触发硬规则，例如标题缺少“微众银行”、直报主体称谓错误、小标题、个案单独成段、正文过短等。

运行：

```bash
uv run --locked pytest tests/test_direct_report_quality_regression.py -q
```

## 自动化测试不覆盖什么

以下问题暂不适合只靠自动化测试判断：

- 是否像正式直报。
- 政策背景是否自然、准确、有必要。
- 是否把微众银行作为全文主体。
- 是否把新闻稿改写成了报送材料。
- 哪些事实该保留、哪些细节该压缩。

这些需要人工评测，后续可以逐步沉淀为更明确的规则。

## 人工评测方式

每次对这 4 个样本跑出当前版本结果后，按下面 6 个维度评估。

评分建议：

```text
0 = 不合格
1 = 勉强可用，但必须大改
2 = 基本可用，需要中等修改
3 = 可用，只需小修
```

| 维度 | 看什么 |
| --- | --- |
| usable | 能不能作为直报初稿继续改 |
| direct_report_style | 是否像直报，而不是新闻稿、宣传稿、摘要 |
| subject_centered | 是否始终以微众银行为主体 |
| policy_connection | 政策背景是否准确、自然、不生硬 |
| fact_accuracy | 是否忠于素材，没有编造事实、数据、结论 |
| revision_priority | 最需要优先改的问题是什么 |

本轮额外发现并修复：

- 用户确认直报与简报称谓规则不同。

### 链接读取

中国证券报链接曾经出现过读取失败，也曾在外部网络权限下读取成功。它是网页读取兜底能力的重点样本。

### 政策挂接

本轮规则：

- 时间节点稿件默认直入主题：案件、活动、获奖、直播、判决等。
- 产品、机制、长期服务、政策落地类素材才允许补充政策库材料。
- 只有用户素材或补充材料明确给出政策文件名称时，正文才可写具体政策名。
- 不确定时，只写宽泛政策导向，或不写政策背景。

### 风险治理材料

第一财经样本涉及“黑灰产”、商标侵权、不正当竞争和消费者权益保护。这个样本重点看：

- 是否从个案判决上升到金融黑灰产治理机制。
- 是否仍像新闻报道。
- 是否按直报口径直接使用“微众银行”，不出现全称、简称引入或“我行”。

### 财经网页噪音

同花顺财经页面可能有行情、广告和推荐阅读噪音。这个样本重点看正文抽取是否干净。

### 转载平台

百家号样本可能存在转载、摘要和平台噪音。这个样本重点看是否读到了有效正文，以及是否能改写成正式直报。

## 后续扩展规则

新增样本时，不要只追求数量。优先补以下类型：

- 写得不理想、需要复盘的失败样本。
- 用户认为方向比较对的好样本。
- 链接读取失败或正文抽取噪音大的样本。
- 政策挂接容易生硬的样本。
- 改稿执行不精准的样本。

新增样本后，必须同步更新：

```text
tests/fixtures/direct_report_quality_cases.json
tests/test_direct_report_quality_regression.py
```
