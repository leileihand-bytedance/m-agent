# 智能审核规则库

> 这是给 LLM 审核员的审核清单。
> 修改这里 → Bot 重启后生效。

---

## 第一层:底线要求

必须过的关,不合格就报错。

### 一、结构识别(先做这步,只定位不审核)

先把文档切成 4 个区域:

| 区域 | 识别方式 | 是否审核 |
|------|---------|---------|
| **文档头** | 段1含"内部资料" | 不审核 |
| **目录(TOC)** | 从"目录"段开始,到"主编/责编"或章节分类前为止 | 目录专项审核 |
| **页脚** | 含"主编/责编"的段 | 不审核 |
| **正文区** | 章节分类之后的所有内容 | 三关审核 + 内容质量审核 |

**5大板块分类**:

| 板块 | 哪些内容应该放这里 |
|------|------------------|
| **党政要闻** | 党和国家领导人（习近平、李强等）活动；国务院其他部委（商务部、外交部、工信部等）动态；全国性重要会议 |
| **监管动态** | **一行一局一会**：中国人民银行、金融监管总局、证监会的新政策、表态、重要会议 |
| **同业动向** | 民营银行（微众银行、网商银行等）、数字银行、虚拟银行相关动态 |
| **市场观察** | 上述4类以外的所有内容（兜底板块）；金融市场的数据、分析、点评；大宗商品、汇率、资本市场动态 |
| **前沿观点** | 分析类文章（政策解读、市场分析等）；一般不判断板块是否正确 |

**注意**：前沿观点板块不做板块归位检查。

**正文区内部结构**(按顺序出现):
```
章节分类(4字:党政要闻/监管动态/同业动向/市场观察/前沿观点)
  新闻标题(5~30字,无句末标点)
  正文(紧跟标题,通常100字+)
  [原文引用](以"原文:"开头) ← 要审核,不是跳过
  (下一个新闻重复上述结构)
```

### 二、正文区三关审核

#### 关卡 1:标题截断 `title-truncated`
- **严重程度**:error
- **定义**:新闻标题被截断,语句本身不完整（不是缩写,而是被砍断了）
- **判断标准**:看标题本身是不是一句完整的话
  - 标题核心词被砍断（如"习近平对朝鲜进"少了"行国事访问"，句子不完整） → **截断**
  - 标题是完整句子,只是比正文更简略 → **不截断**（只是缩写，不是截断）
- **典型情况**:
  - ❌ 标题:"习近平对朝鲜进" → 正文:"6月8日,国家主席习近平抵达平壤..." → 标题"进"之后被砍断,语句不完整
  - ❌ 标题:"中东局势骤然升级,国际油" → 正文:"中东地缘冲突骤然升级。伊朗向以色列发射导弹..." → 标题"国际油"之后被砍断
  - ❌ 标题:"国务院政策例行吹风会介" → 正文:"国务院新闻办公室举行发布会..." → 标题被截断
  - ❌ 标题:"习近平在二十届中央纪委五次全会" → 正文:"1月12日...习近平在...全体会议上发表重要讲话..." → 标题"全会"后无谓语,核心动作"发表重要讲话"被截断,语句不完整
  - ✅ 标题:"李强主持召开国务院常务会议,研究审计查出问题整改工作" → 正文:"6月11日,国务院总理李强主持召开国务院常务会议..." → 标题是完整句子,只是比正文简略
  - ✅ 标题:"SpaceX创史上最大IPO" → 正文:"SpaceX于6月12日在纳斯达克上市..." → 标题完整
- **注意**:标题比正文简略是正常的（正常缩写）；只有标题本身语句不完整才算截断

---

#### 关卡 2:标题-正文错配 `content-mismatch`
- **严重程度**:error
- **定义**:标题说的"事"和正文说的"事"是**完全不同**的两件事
- **判断标准**:标题的核心事件是什么?正文的的核心事件是什么?是同一件吗?
  - **同一件事**:同一会议、同一文件、同一人物活动、同一事件 → 不报错配
  - **不同的事**:A会议 vs B文章、A人物 vs B人物 → 报错配
- **⚠️ 会议名称必须精确匹配**:标题提到的会议名称（如"金融稳定工作会议"、"国库工作会议"）必须与正文开头提到的会议名称完全一致。如果标题说"金融稳定工作会议"但正文开头说"国库工作会议"，**必须报错配**。
- **典型情况**:
  - ❌ 标题:"国务院政策例行吹风会介绍推进城市更新工作有关情况"
    正文:"第11期《求是》杂志将发表习近平总书记重要文章《前瞻布局和发展未来产业》"
    → 标题在说"政策吹风会",正文在说"求是杂志文章",**完全不同**,错配
  - ❌ 标题:"丁向群同志任国家金融监督管理总局党委书记"
    正文:"中国人民银行、国家档案局联合发布《中国人民银行档案管理规定》"
    → 标题在说"人事任命",正文在说"档案制度",**完全不同**,错配
  - ❌ 标题:"中国人民银行召开2026年金融稳定工作会议"
    正文:"3月20日，中国人民银行召开2026年国库工作会议..."
    → 标题"金融稳定"，正文"国库"，**完全不同**，错配
  - ❌ 标题:"中国人民银行召开2026年国库工作会议"
    正文:"近日，中国人民银行召开2026年金融稳定工作会议..."
    → 标题"国库"，正文"金融稳定"，**完全不同**，错配
  - ✅ 标题:"SpaceX创史上最大IPO"
    正文:"SpaceX于6月12日在纳斯达克上市,以每股135美元发行5.556亿股,融资750亿美元..."
    → 同一件事(IPO),不报错配
  - ✅ 标题:"第11期《求是》杂志将发表习近平总书记重要文章"
    正文:"第11期《求是》杂志将发表习近平总书记重要文章《前瞻布局和发展未来产业》..."
    → 同一篇文章,正文是标题的展开,不报错配

#### 关卡 3:正文完整性 `content-incomplete`
- **严重程度**:error
- **定义**:正文段在句中戛然而止,语义不完整,缺宾语或结束语
- **判断标准**:
  - 段末有完整句末标点(。!？) → 基本完整
  - 段末无标点,语义在句中突然结束 → 截断
  - 段末有标点但语义明显没说完("...,推动多渠道。" → 缺"XX发展"之类) → 截断
- **典型情况**:
  - ❌ "...推动多渠道。" → 无宾语,戛然而止
  - ❌ "...但居民存款减少,除了" → "除了"后面没了,语义不完整
  - ❌ "...成为资本" → "成为资本..."后面没了
  - ✅ "...推动多渠道发展。" → 有宾语"发展",完整
  - ✅ "...较3月增加2.6万亿元。" → 有完整谓语 and 宾语,完整
- **注意**:正文区所有段落都要检查完整性,包括"原文:"开头的引用段

### 三、目录(TOC)专项审核

**只针对目录区域**(从"目录"段到"主编/责编"或章节分类前为止)

#### 规则 1:目录项不带序号 `toc-no-ordinal`
- **严重程度**:error
- **定义**:目录项(章节名、新闻标题)不应该带"一、二、三、四"序号
- **反例**:
  - ❌ "一、党政要闻"
  - ❌ "二、监管动态"
  - ❌ "三、市场观察"
- **正例**:
  - ✅ "党政要闻"
  - ✅ "监管动态"
  - ✅ "市场观察"
- **注意**:页码数字(如末尾的"2""3""4")不算序号,不用管

---

#### 规则 2:目录序号不跳号 `toc-seq-skip`
- **严重程度**:error
- **定义**:如果目录有编号,序号必须连续;跳过某个序号是错误
- **反例**:
  - ❌ 出现"三、"但前面没有"二、"(跳号)
- **正例**:
  - ✅ "一、二、三、四" 连续编号
  - ✅ 没有编号(纯章节名)也行

---

#### 规则 3:目录与正文对得上 `toc-mismatch`
- **严重程度**:error
- **定义**:目录里的章节名、新闻标题、排列顺序,必须与正文区对应得上
- **检查三项**:
  1. **章节名一致**:目录里出现的章节名,正文里是否都有对应?
  2. **新闻标题一致**:目录里列出的新闻标题,正文里是否都有对应?(允许标题被截断的情况,只要能识别出是同一件事)
  3. **顺序一致**:目录里章节和标题的排列顺序,与正文区的排列顺序是否一致?
- **典型情况**:
  - ✅ 目录:党政要闻 → 习近平访朝 → 李强主持常务会议 → 监管动态 → ...
    正文:党政要闻(分类) → 习近平访朝(标题) → 李强主持常务会议(标题) → 监管动态(分类) → ...
    → 三项都对得上,没问题
  - ❌ 目录顺序与正文顺序不一致(如目录先列李强再列习近平,但正文先出现习近平再出现李强) → 顺序对不上

### 四、全局格式规则(适用于文档所有区域)

#### `quote-pair` 引号不成对
- **严重程度**:warning
- **定义**:引号("""《》『』)必须成对,不能只开不闭或只闭不开
- **典型情况**:
  - ❌ "《规定》明确..." → 只开了《没闭》
  - ✅ "《规定》明确...》" → 配对正确

#### `num-unit` 数字单位格式
- **严重程度**:warning
- **定义**:数字和单位之间一般不加空格(除非是英文数字)
- **典型情况**:
  - ❌ "A 股" "3 万亿元" → 空格多余
  - ✅ "A股" "3万亿元" → 正确
  - ✅ "5.39%" "0.9%" → 正确(数字+英文单位)

#### `mixed-punct` 中英文标点混用
- **严重程度**:warning
- **定义**:中文句子里不应出现英文标点(如中文句子里用英文逗号,)
- **典型情况**:
  - ❌ "你好,世界" → 中文句子里用英文逗号
  - ✅ "你好，世界" → 正确的中文逗号

## 第二层:内容质量要求

这一层判断内容**好不好**,不属于底线,但明显有问题的要报出来。

### 规则 5:不该放的 `content-out-of-scope`
- **严重程度**:error
- **定义**:跟银行经营管理、宏观经济政策**完全无关**的内容,不该出现在周报里
- **判断标准**:看这段内容的核心主题是什么?
  - 与银行经营管理完全无关(如娱乐、天气、个人生活)
  - 与宏观经济政策完全无关
  - 普通科技公司动态（与金融无关）
- **典型情况**:
  - ❌ 娱乐新闻(如某明星演唱会、电影上映)
  - ❌ 纯生活信息(如天气预报、餐厅推荐)
  - ❌ 与金融/银行/经济政策毫无关联的八卦新闻
  - ❌ 某互联网公司发布与金融无关的产品（如某公司发布新手机）
- **应该放的**:
  - 涉及银行、监管、金融市场、企业经营、政策解读的内容 → 应该放
  - 涉及金融科技、数字银行、央行数字货币等 → 应该放
  - 房地产调控政策（与宏观经济密切相关） → 应该放
  - 涉及科技但跟数字金融、金融科技相关的 → 应该放

---

### 规则 6:明显放错板块 `content-wrong-section`
- **严重程度**:error
- **定义**:内容**明显**属于某个板块,但被放到了其他板块（前沿观点除外）
- **判断标准**:先判断内容来源/主体属于哪个板块，再看实际被放在哪个板块
  - 央行/金融监管总局/证监会 → 监管动态
  - 党和国家领导人/国务院其他部委 → 党政要闻
  - 民营银行/数字银行/虚拟银行 → 同业动向
  - 资本市场综述/股市综述/债市评论/汇市评论/A股综述/港股综述/美股综述/大宗商品 → **市场观察**（不得放入监管动态；前沿观点除外）
  - 其他所有内容 → 市场观察（不报错）
- **典型情况**:

| 内容来源/主体 | 实际放的板块 | 正确板块 | 判断 |
|-------------|-----------|---------|------|
| 央行发布政策 | 市场观察 | 监管动态 | ✅ 报错 |
| 党和国家领导人活动 | 市场观察 | 党政要闻 | ✅ 报错 |
| 证监会重要发言 | 市场观察 | 监管动态 | ✅ 报错 |
| 金融监管总局会议 | 党政要闻 | 监管动态 | ✅ 报错 |
| 民营银行动态 | 党政要闻/市场观察 | 同业动向 | ✅ 报错 |
| **资本市场综述/A股综述/港股综述/美股综述/债市汇市评论** | **监管动态** | **市场观察** | ✅ **报错** |
| A股/港股/美股行情综述（如"本周A股主要指数集体走强"） | 监管动态下的正文段 | 市场观察 | ✅ **报错** |
| 金融科技公司动态 | 党政要闻 | 市场观察 | ⚠️ 不报错（模糊） |

---

### 规则 7:重复内容 `content-duplicate`
- **严重程度**:error
- **定义**:同一件事在周报里出现了**两次或以上**(含不同来源的同一事件)
- **判断标准**:
  - 标题不同但说的是**同一件事** → 重复
  - 同一篇报道/公告被多处来源收录 → 重复
  - 完全不同的内容 → 不重复
- **典型情况**:
  - ❌ 标题A:"SpaceX创史上最大IPO" + 标题B:"SpaceX在纳斯达克上市" → 同一件事,重复
  - ❌ 标题A:"央行宣布降准" + 标题B:"央行下调存款准备金率" → 同一件事,重复
  - ✅ 标题A:"腾讯发布财报" + 标题B:"阿里发布财报" → 不同公司,不重复
  - ✅ 标题A:"特斯拉在美降价" + 标题B:"特斯拉在华降价" → 不同市场,可视为不同事件
- **注意**:
  - 同一事件的多角度报道(如一个政策出台+一个政策解读) → 不算重复,是正常的多篇
  - 是否重复要看**核心事件**是否相同,不是看关键词是否相同

---

### 规则 8:过时信息 `content-outdated`
- **严重程度**:error
- **定义**:内容引用的事件**明显早于**周报的时间范围
- **判断标准**:
  - 周报是6月20日发布的,收录的是6月13日-6月19日的重点信息
  - 如果正文提到"5月的CPI"、"4月的会议"、"去年的政策"等 → 可能是过时信息
  - 要结合上下文判断:是正文中引用了旧数据,还是整篇内容都是旧闻?
    - 引用旧数据(如"5月CPI同比上涨0.3%")作为论证支撑 → 一般不算过时
    - 整篇内容在说一件已经发生很久的事,没有近期动态 → 过时
- **典型情况**:
  - ❌ 正文:"据悉,上述会议于4月15日召开..." → 周报时间是6月,4月的事属于过去
  - ❌ 正文:"此前,有消息称..." → 没有具体日期,但语境明显是旧闻
  - ⚠️ 正文:"据5月份数据..." → 如果是引用旧数据作论证,不一定是问题;如果是说一件发生了一段时间的事,可能是过时
  - ✅ 正文:"6月18日,央行宣布..." → 时效性正确,不是过时
  - ✅ 正文:"会议指出,当前要..." → 没有时间暗示,属于常态化表述,不报错

## 输出格式

**只输出 JSON,不要任何解释**:

```json
{"issues": [
  {"paragraph_index": 22, "rule_id": "title-truncated", "description": "标题被截断,缺少'行国事访问'", "original_text": "习近平对朝鲜进"},
  {"paragraph_index": 27, "rule_id": "content-mismatch", "description": "标题说政策吹风会,正文说求是杂志文章,完全不同", "original_text": "国务院政策例行吹风会介绍推进城市更新工作有关情况"},
  {"paragraph_index": 34, "rule_id": "content-incomplete", "description": "正文段末'推动多渠道。'语义不完整,缺宾语", "original_text": "...推动多渠道。"},
  {"paragraph_index": 40, "rule_id": "content-out-of-scope", "description": "内容与银行经营管理、宏观经济政策完全无关,娱乐新闻", "original_text": "某明星演唱会圆满落幕"},
  {"paragraph_index": 45, "rule_id": "content-wrong-section", "description": "内容明显属于监管动态(央行降准),却被放在了市场观察", "original_text": "央行宣布降准0.25个百分点"}
]}
```

- `paragraph_index` 从 0 开始
- `rule_id` 必须是本规则库里列出的 ID
- `original_text` 必须是该段完整原文
- 如果没有问题: `{"issues": []}`
